Descrizione del Progetto:

Il Progetto riguarda la creazione di un'applicazione web per studenti.
La struttura dell'applicazione è di un semplice registro elettronico, con l’aggiunta di una sezione chat dove gli studenti possono parlare con i professori oppure con i compagni di classe;
inoltre è presente anche una chat-room per le comunicazioni importanti (avvisi scolastici, circolari e comunicazioni dei professori), i messaggi mandati in questa chat-room vengono inoltrati via SMS a tutti i partecipanti per facilitare le comunicazioni urgenti.
L’applicazione è pensata per una classe sola; gli utenti accedono utilizzando l'autenticazione di google, inserendo la propria email e password.
Una volta eseguito l’accesso verranno indirizzati alla homepage dove si trovano le notizie e le informazioni principali. L’invio dei messaggi viene fatto attraverso Socket IO e tutte le comunicazioni vengono salvate sul DB
attraverso un collegamento fra Node js e MSSQL.
Le notifiche SMS vengono gestite da un servizio di Rest Api chiamato INFOBIP.



Servizi utilizzati:


  • Angular

  • Api Google

  • Bootstrap

  • Socket.io

  • Mssql

  • InfoBip


Problemi Riscontrati nello sviluppo del progetto:

Per quanto le piattaforme di coding online siano utili, le cosiddette “ready-to-code” non sono uno strumento ottimale per sviluppare un applicazione.
Gitpod per esempio, richiede molto tempo per avviarsi e bisogna aspettare che vengano caricate tutte le modifiche fatte in precedenza.
Inoltre non lavorando in locale, quando si avvia un server Node, questo non avrà il solito indirizzo “Localhost” ma uno generato da gitpod; questo indirizzo cambia ogni volta che si avvia Gitpod, quindi se ci sono richieste per il server attraverso l’url, bisogna aver premura di cambiare questo indirizzo ogni volta.

Visto che gli indirizzi cambiano ogni volta che si avvia Gitpod, in google console per implementare l'autenticazione bisogna acconsentire ogni volta per i nuovi indirizzi le “Origini JavaScript autorizzate”.

InfoBip pur essendo un servizio api gratuito, nella versione di prova non permette di mandare più di 50 sms per account (per mandarne altri bisogna pagare).
InfoBip nella versione di prova non permette di registrare più di 2 numeri nel loro sistema. Non è possibile mandare messaggi a numeri non registrati, per cui si possono mandare sms solo a quei 2 numeri registrati.

L’integrazione fra grafica e parte server dell'applicazione è difficile; è meglio concentrarsi prima sullo sviluppo delle funzionalità e della parte del server e dopo aggiungere la grafica.
Facendo le due cose insieme si perde tempo, invece farle separatamente e poi integrarle è molto complicato.

Google Chrome blocca automaticamente il popup della finestra di autenticazione, per il browser non è considerato attendibile perché non è un indirizzo registrato. Per cui ho dovuto usare Microsoft Edge per sviluppare l’applicazione.
